# Wa7at ElDr3yah Booking & Financial Management System

## نظام واحة الدرعية لإدارة الحجوزات والحسابات المالية

A full-stack booking and financial management system designed to manage reservations, track revenues and expenses, calculate remaining payments, monitor profit/loss, and generate monthly financial reports.

نظام Full Stack لإدارة الحجوزات والحسابات المالية، الهدف منه تحويل الشغل اليدوي الموجود في Excel إلى System منظم وسهل الاستخدام.

---

## Project Overview | نظرة عامة

Wa7at ElDr3yah System is built for businesses that manage reservations manually, such as rest houses, event halls, small venues, and booking-based services.

The system allows internal admins to register bookings, track paid amounts, calculate remaining balances automatically, record expenses, manage capital, and generate financial reports.

النظام معمول لإدارة الحجوزات ومتابعة الحسابات بشكل داخلي، بدون تسجيل عملاء أو حسابات عامة. الاستخدام الأساسي بيكون من خلال Admins و Owner فقط.

---

## Business Problem | المشكلة التي يحلها النظام

Before this system, bookings and financial tracking were handled manually using Excel sheets.

This caused problems such as:

- Difficulty tracking booked dates
- Possibility of double booking
- Manual calculation errors
- Unclear remaining balances
- Hard tracking of revenues and expenses
- No structured monthly financial reports
- No clear profit/loss calculation

النظام بيحل المشاكل دي عن طريق توفير Backend API وواجهة أمامية لإدارة كل العمليات من مكان واحد.

## Main Business Logic | منطق العمل الأساسي

### 1. Booking Registration

Admin creates a booking manually by entering:

- Customer name
- Contact number
- Booking date
- Day name
- Booking type/package
- Total price
- Paid amount
- Notes

The system automatically calculates:

```txt
Remaining Amount = Total Price - Paid Amount
```

The admin does not enter the remaining amount manually.

---

### 2. Prevent Double Booking

The system prevents booking the same date more than once.

Before creating or updating a booking, the system checks:

```txt
If another booking exists on the same date → reject the request
```

Example:

```txt
If 2026-05-01 is already booked,
another booking on 2026-05-01 will be rejected.
```

---

### 3. Revenue Calculation

Every paid amount in bookings is treated as revenue.

```txt
Total Revenue = Sum of all PaidAmount values
```

---

### 4. Expense Tracking

Admins can manually add expenses such as:

* Maintenance
* Cleaning
* Supplies
* Staff costs
* Other operational costs

Each expense contains:

* Title
* Amount
* Category
* Expense date
* Notes
* Created by user

---

### 5. Capital Management

The system allows recording the available capital.

Capital is used as a financial reference to compare revenues, expenses, and business performance.

---

### 6. Profit and Loss Calculation

The system calculates profit/loss using:

```txt
Net Profit Or Loss = Total Revenue - Total Expenses
```

If the result is positive:

```txt
Profit
```

If the result is negative:

```txt
Loss
```

Profit/Loss percentage can also be calculated to show financial performance.

---

### 7. Monthly Reports

The system supports monthly financial reports.

Each report includes:

* Month
* Year
* Total revenue
* Total expenses
* Net profit or loss
* Profit/loss percentage
* PDF file path
* Generated by user
* Generated date

Reports are designed to be exported as PDF.

---

### 8. Authentication and Authorization

The system uses JWT Authentication.

Only internal users can access the system.

Roles:

```txt
Owner
Admin
```

Admins can manage bookings and financial operations.

Owners can manage users/admins and have higher-level control.

---

### 9. Login Using Email

Users log in using:

```txt
Email + Password
```

Passwords are not stored as plain text.

They are hashed using BCrypt.

---

### 10. Forgot Password Flow

The system supports forgot password using email.

Flow:

```txt
User enters email
↓
System generates reset code
↓
System hashes the reset code
↓
Code is sent to email
↓
User enters code and new password
↓
System verifies the hashed code
↓
Password is updated
```

Security features:

* Reset code expires after 10 minutes
* Reset code is stored as hash
* Rate limit prevents requesting multiple codes too quickly
* New password must be strong

## Setup and Run

### 1. Clone Repository

```bash
git clone <repository-url>
```

### 2. Configure appsettings.json

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=.\\MSSQLSERVER01;Database=wa7t_eldr3ya;Trusted_Connection=True;TrustServerCertificate=True;MultipleActiveResultSets=true"
  },

  "Jwt": {
    "Key": "YOUR_LONG_SECRET_KEY",
    "Issuer": "Wa7at_ElDr3yah_API",
    "Audience": "Wa7at_ElDr3yah_Client",
    "DurationInMinutes": 60
  },

  "EmailSettings": {
    "From": "your_email@gmail.com",
    "SmtpServer": "smtp.gmail.com",
    "Port": 587,
    "Username": "your_email@gmail.com",
    "Password": "your_app_password"
  }
}
```

---

### 3. Run Migration

```powershell
Add-Migration InitialCreate
Update-Database
```

Or:

```bash
dotnet ef migrations add InitialCreate
dotnet ef database update
```

---

### 4. Run Backend

```bash
dotnet run
```

Swagger:

```txt
https://localhost:7189/swagger
```

### 5. Run Frontend

```bash

```
